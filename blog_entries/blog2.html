CodeQuest Devlog #2: Learning, Testing and Playing

Small Wins, Big Learning

5 min read

June 25, 2025

I’ve officially started learning Unity and C# from scratch this week, and while it hasn’t been easy, it’s already been rewarding.

I’ve spent the last two weeks slowly learning about C# and the basics of game development. I’ve planned a lot, from the game design document to searching for the right assets that fit the genre of my game. I also dabbled a bit in Unity to familiarize myself with the user interface, which I find a tad difficult or at least very different, and explored some of the basic functions.

Last week was all about baby steps. It feels like I’m learning how to walk again. Thankfully, one thing I noticed is how much Java has helped me pick up C# faster. It’s like having a map in a new country.

I wrote my first script in Unity with C#, and it was not as bad as I thought. I watched a couple of videos, followed some tutorials, and decided to remake the famous game Flappy Bird. I have been building a scene, writing a few scripts, learning about GameObjects, and testing the game. I’ve really been enjoying the journey so far.

I faced a few challenges over the past weeks. I changed computers, and setting everything up on my new one took some time. Unity runs much more smoothly now, and the graphics look great. On top of that, I had to study for an exam and juggle other projects.

Next Week's Plan:

  • Build the main Unity scene
  • Finish my Flappy Bird clone game
  • Work on the core gameplay
  • Finish the game design document

I’m excited to keep pushing forward. I’ll be sharing more updates as I continue this journey. Stay tuned for the next devlog!
